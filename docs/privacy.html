Политика конфиденциальности

Навык «Google Calendar» для Яндекс Алисы  ·  Последнее обновление: 17 июля 2026 г.

Что делает навык

Навык «Google Calendar» позволяет создавать события в вашем Google Календаре голосовыми командами через Яндекс Алису.

Какие данные Google аккаунта используются

С вашего разрешения навык получает доступ к Google Календарю через стандартный протокол OAuth 2.0 с областью действия https://www.googleapis.com/auth/calendar.events.owned.

Приложение получает доступ к следующим данным пользователя Google:

  • События Google Календаря — навык может создавать новые события в вашем основном календаре. Навык не читает, не изменяет и не удаляет существующие события.

Никакие другие данные Google (имя, email, контакты, файлы Drive и т.д.) не запрашиваются и не используются.

Как используются данные Google

Данные вашего аккаунта Google используются исключительно для создания новых событий в вашем Google Календаре по вашей голосовой команде. Никакой иной обработки, анализа, продажи или передачи данных не производится. Приложение действует строго как посредник между вашей голосовой командой и API Google Calendar.

Хранение данных

Навык не хранит ваши токены, персональные данные или содержимое календаря. OAuth-токен передаётся платформой Яндекс Диалоги непосредственно в момент запроса и нигде не сохраняется разработчиком.

Сроки хранения и удаление данных

Поскольку навык не сохраняет никаких пользовательских данных, срок хранения равен нулю: OAuth-токен существует только в оперативной памяти на время обработки одного запроса и автоматически уничтожается по его завершении.

Если вы хотите полностью прекратить доступ навыка к вашему аккаунту Google, отзовите авторизацию в настройках Google: myaccount.google.com/permissions. После отзыва никакие данные, связанные с вашим аккаунтом, у разработчика не остаются — удалять нечего.

По любым запросам, связанным с удалением данных, обращайтесь на alicecalendarassistant@gmail.com — ответ будет дан в течение 30 дней.

Защита данных

Поскольку навык не хранит никаких пользовательских данных, поверхность атаки сведена к минимуму. Тем не менее на каждом уровне применяются меры защиты:

  • Шифрование при передаче. Все запросы между Яндекс Алисой, навыком и Google Calendar API выполняются исключительно по HTTPS/TLS — данные не могут быть перехвачены в открытом виде.
  • Минимальные права доступа. Запрашивается единственная область OAuth calendar.events.owned — наименьший набор прав, необходимый для создания событий.
  • Отсутствие хранилища. OAuth-токен никогда не записывается на диск и не сохраняется в базе данных, что исключает его утечку из систем разработчика.
  • Краткосрочные токены. Используются только краткосрочные токены доступа, выдаваемые Яндекс Диалогами на каждый запрос; долгосрочные токены обновления разработчиком не хранятся.

Передача данных третьим лицам

Никакие ваши данные не передаются третьим лицам. Единственный внешний сервис, с которым взаимодействует навык, — Google Calendar API от имени вашего аккаунта.

Управление доступом

Вы можете отозвать доступ навыка в любой момент в настройках аккаунта Google: myaccount.google.com/permissions .

Контакт

По вопросам конфиденциальности пишите на alicecalendarassistant@gmail.com.

Privacy Policy

«Google Calendar» skill for Yandex Alice  ·  Last updated: July 17, 2026

What the skill does

The «Google Calendar» skill lets you create events in your Google Calendar using voice commands via Yandex Alice.

What Google user data is accessed

With your permission, the skill accesses Google Calendar through the standard OAuth 2.0 protocol with the scope https://www.googleapis.com/auth/calendar.events.owned.

The app accesses the following Google user data:

  • Google Calendar events — the app can create new events in your primary Google Calendar. It cannot read, modify, or delete existing events.

No other Google user data (name, email address, contacts, Drive files, etc.) is requested or accessed.

How we use Google user data

Google account data is used exclusively to create new events in your Google Calendar in response to your voice commands. No other processing, analysis, sale, or transfer of this data takes place. The application acts strictly as an intermediary between your voice command and the Google Calendar API.

Data storage

The skill does not store your tokens, personal data, or calendar content. The OAuth token is provided by the Yandex Dialogs platform on each request and is never persisted by the developer.

Data retention and deletion

Because the skill stores no user data, the retention period is zero: the OAuth token exists only in memory for the duration of a single request and is automatically discarded once the request is complete.

If you wish to fully terminate the skill's access to your Google account, revoke the authorization in your Google Account settings: myaccount.google.com/permissions. After revoking, no data associated with your account remains with the developer — there is nothing left to delete.

For any data deletion requests, contact alicecalendarassistant@gmail.com — you will receive a response within 30 days.

Data protection

Because the skill stores no user data, the attack surface is minimal by design. The following security measures are in place at every layer:

  • Encryption in transit. All communication between Yandex Alice, the skill, and the Google Calendar API is conducted exclusively over HTTPS/TLS, ensuring data cannot be intercepted in plain text.
  • Least-privilege access. Only the single OAuth scope calendar.events.owned is requested — the minimum permission required to create events.
  • No data at rest. The OAuth token is never written to disk or stored in any database, eliminating the risk of a credential leak from the developer's infrastructure.
  • Short-lived tokens. Only short-lived access tokens issued by Yandex Dialogs per request are used; no long-lived refresh tokens are retained by the developer.

Third-party sharing

No data is shared with third parties. The only external service the skill communicates with is the Google Calendar API, acting on behalf of your account.

Revoking access

You can revoke the skill's access at any time in your Google Account settings: myaccount.google.com/permissions .

Contact

For any privacy-related questions, contact alicecalendarassistant@gmail.com.
